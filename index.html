Uninstall

To completely remove PersistenceAI from your system:

Windows (PowerShell)

Completely remove PersistenceAI including binaries, cache, and config files (handles file locks automatically)

iwr -useb https://persistence-ai.github.io/Landing/uninstall.ps1 | iex

Linux / macOS

Completely remove PersistenceAI including binaries, cache, and config files

# Stop running processes
pkill -f pai 2>/dev/null; pkill -f persistenceai 2>/dev/null
sleep 2
# Remove all directories
rm -rf ~/.persistenceai ~/.pai ~/.config/persistenceai ~/.config/pai ~/.local/share/persistenceai ~/.local/share/pai 2>/dev/null
# Remove from PATH in shell config files
for file in ~/.bashrc ~/.zshrc ~/.bash_profile ~/.profile; do
  [ -f "$file" ] && sed -i.bak "\\|.persistenceai/bin|d; \\|.pai/bin|d" "$file" 2>/dev/null
done
echo "PersistenceAI completely uninstalled"

Configure

PersistenceAI is highly configurable. Customize tools, agents, models, themes, keybinds, formatters, permissions, and more through your PersistenceAI.json config file.

Providers & Authentication

Configure API keys for any LLM provider. PersistenceAI supports Claude, OpenAI, Google, and local models via Ollama.

# Run in TUI /connect # Or via CLI persistenceai auth login # Or use environment variables export OPENAI_API_KEY="your-key-here" export ANTHROPIC_API_KEY="your-key-here"

Tools

Control which tools the LLM can use. Configure globally or per-agent. Built-in tools include:

bash

Execute shell commands

edit

Modify existing files

write

Create new files

read

Read file contents

grep

Search with regex

glob

Find files by pattern

webfetch

Fetch web content

patch

Apply patches

# Configure tools globally { "tools": { "write": true, "bash": true, "webfetch": false }, "agent": { "plan": { "tools": { "write": false, "bash": false } } } }

Rules (AGENTS.md)

Provide custom instructions to PersistenceAI by creating an AGENTS.md file. This helps PersistenceAI understand your project structure and coding patterns.

# Initialize AGENTS.md /init # Or create manually # Project-specific: ./AGENTS.md # Global: ~/.config/PersistenceAI/AGENTS.md

Formatters

PersistenceAI automatically formats code using language-specific formatters. Supports Prettier, Biome, Ruff, gofmt, and many more.

{ "formatter": { "prettier": { "disabled": false }, "custom-formatter": { "command": ["npx", "prettier", "--write", "$FILE"], "extensions": [".js", ".ts"] } } }

Permissions

Control what actions agents can take. Set permissions to ask, allow, or deny for file edits, bash commands, and web fetching.

{ "permission": { "edit": "ask", "bash": { "git push": "ask", "*": "allow" } } }

MCP Servers

Extend PersistenceAI with Model Context Protocol servers. Add database access, API integrations, and third-party services.

Out-of-the-box enabled MCPs:

exa

Remote web search via Exa AI for current information

morph

Local code search and transformation tool

context7

Documentation search for official library docs

grep_app

Code search across millions of GitHub repositories

{ "mcp": { "my-mcp-server": { "type": "local", "command": ["npx", "-y", "@modelcontextprotocol/server-everything"], "enabled": true } } }

LSP Servers

Configure Language Server Protocol servers for IDE-like features. PersistenceAI automatically downloads and manages LSP servers.

{ "lsp": { "typescript": { "command": ["typescript-language-server", "--stdio"] } } }

Custom Tools

Define your own tools that the LLM can call. Create custom functions for project-specific workflows.

{ "customTools": { "deploy": { "description": "Deploy the application", "command": ["npm", "run", "deploy"] } } }

Config File Locations

PersistenceAI supports multiple config file locations with merging precedence. Configs are merged, not replaced.

# Global config (applies to all projects) ~/.config/PersistenceAI/PersistenceAI.json # Project-specific config (overrides global) ./PersistenceAI.json # Custom config path via environment variable export PersistenceAI_CONFIG=/path/to/config.json

Modes

Create custom modes with specific temperature, prompts, and tool configurations for different workflows.

{ "mode": { "analyze": { "temperature": 0.1, "prompt": "{file:./prompts/analysis.txt}", "tools": { "write": false, "bash": false } }, "brainstorm": { "temperature": 0.7, "prompt": "{file:./prompts/creative.txt}" } } }

Custom Commands

Define reusable commands for repetitive tasks. Commands can have templates, descriptions, and agent assignments.

{ "command": { "test": { "template": "Run the full test suite with coverage report.", "description": "Run tests with coverage", "agent": "build", "model": "anthropic/claude-haiku-4-5" }, "component": { "template": "Create a new React component named $ARGUMENTS with TypeScript.", "description": "Create a new component" } } }

Sharing

Configure how conversations are shared. Options: manual (default), auto, or disabled.

{ "share": "manual" // "auto" - Automatically share new conversations // "disabled" - Disable sharing entirely }

Environment Variables & File References

Use environment variables and file contents in your config for sensitive data and dynamic configuration.

{ "model": "{env:PersistenceAI_MODEL}", "provider": { "anthropic": { "options": { "apiKey": "{env:ANTHROPIC_API_KEY}" } }, "openai": { "options": { "apiKey": "{file:~/.secrets/openai-key}" } } }, "instructions": ["CONTRIBUTING.md", "docs/guidelines.md"] }

Frequently Asked Questions

What is PersistenceAI?

PersistenceAI (also known as PAI) is an enterprise-grade terminal AI IDE and agentic coding multiplexer. It provides full IDE features with multi-agent AI integration, multi-session AI conversations, terminal-first design, and vibe coding capabilities.

What terminal do I need?

PersistenceAI requires a modern terminal emulator that supports TUI (Terminal User Interface). Recommended terminals include WezTerm, Alacritty, Ghostty, Kitty, or Windows Terminal. On Windows, PowerShell 5.1 won't workâ€”use Windows Terminal or PowerShell 7+.

Which LLM providers are supported?

PersistenceAI supports dozens of providers through Models.dev, including OpenAI, Anthropic (Claude), Google (Gemini), and local models via Ollama. You can also configure custom providers with OpenAI-compatible APIs.

Can I use PersistenceAI without an API key?

Yes! You can use local models with Ollama. Install Ollama, download a model, and configure PersistenceAI to use it. No API keys required for local development.

How does multi-session work?

You can run multiple AI conversations simultaneously in separate sessions. Each session maintains its own context and history. Create new sessions with <leader>N (default: Ctrl+X then N) or list/switch sessions with <leader>L (default: Ctrl+X then L). Perfect for working on different features or tasks in parallel.

Does PersistenceAI work over SSH?

Yes! PersistenceAI is designed for remote development. Simply SSH into your remote server and run PersistenceAI. All IDE features work over SSH, making it perfect for cloud instances and remote servers.

Can I customize the keybindings?

Yes! PersistenceAI supports fully customizable keybindings through your config file. You can remap any command to your preferred key combination. See the Configure section for details.

What's the difference between Build, Plan, and Oligarchy agents?

The Plan agent creates a step-by-step plan before making changes, perfect for complex features. The Build agent directly implements changes, ideal for straightforward tasks. The Oligarchy agent combines both with a voting systemâ€”both Build and Plan propose solutions and vote on each other's proposals, ensuring consensus before execution. You can switch between them with Tab or create custom agents for specific workflows.

How do I share a session with my team?

Use the /share command in the TUI to generate a shareable link. This creates a read-only view of your conversation that others can access via web browser. Perfect for code reviews, debugging, or sharing AI-generated solutions.

Is PersistenceAI open source?

PersistenceAI is not fully open source, though some areas may be open sourced in the future. The project is actively developed and available for use. Check our GitHub repository for the latest licensing information.

Can I use PersistenceAI in CI/CD pipelines?

Yes! PersistenceAI's server mode is perfect for CI/CD. Start a headless server and access it via API or attach a client. It also integrates directly with GitHub Actionsâ€”mention /persistenceai in issues or PRs to trigger AI assistance.
